## Description

Kellogg holds both the 1997 version of the Emerging Markets Data Base (EMDB), Monthly Stock Series, from the [International Finance Corporation (IFC)](http://www.ifc.org) and the 2001 Standard & Poor's version. The IFC version has data up to October 1997; the 2001 release includes data up to May 2001.

EMDB covers emerging and "frontier" stock markets. It includes market performance, indicators, market capitalization, monthly value traded, local stock market price indexes, and prices in both local currency and US dollars.

The database includes two series of indexes:

- **IFC Global (IFCG) indexes** — reflect restrictions on foreign ownership
- **IFC Investible (IFCI) index series** — reflects new opportunities for investment since the mid-1980s

These indexes are calculated for prices and returns. Depending on the country, annual coverage starts as early as 1975, while monthly coverage starts in 1986.

Northwestern University Library subscribes to the "Emerging Stock Markets Factbook," the corresponding print edition containing the most recent data (call number 332.6405 E 53 in Government Documents stacks).

## Access

Current Kellogg faculty, staff, and students may access this database through the Windows server **ksmr01.kellogg.local**:

1. Log in to the KELLOGG domain. If using a Windows XP laptop outside the domain, type `kellogg.local\your_netid` when prompted.
2. If your computer uses M as a drive letter for another device, change that drive letter first using Computer Management under Administrative Tools (requires reboot).
3. Select **Run** from the Start menu, type `\\ksmr01.kellogg.local`, and click OK. Open the **appshortcuts** folder and double-click the **EMDB** shortcut. On Windows Vista, right-click the EMDB icon and choose **Run as Administrator**.

**Troubleshooting:**

- *"Could not find FoxPro files ODBC driver"* — The program will still run but cannot connect directly to Excel or produce graphics. Use the ODBC folder in appshortcuts for the driver installer and installation instructions.
- *Blue screen crash* — This may be caused by a Symantec Antivirus bug. Disable network-drive scanning: open Symantec Antivirus → Configure → File System Auto-Protect → uncheck **Enable Scanning** under Network Scanning Options.
